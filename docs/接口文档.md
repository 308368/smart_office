> 版本：V1.0

## 一、通用说明

### 1.1 认证方式

- 所有接口需要在请求头中携带 Token
- 格式：`Authorization: Bearer <token>`
- 登录接口除外

### 1.2 统一响应格式

```json
{
    "code": 200,
    "msg": "success",
    "data": {},
    "timestamp": 1711000000000
}
```

### 1.3 分页响应格式

```json
{
    "code": 200,
    "msg": "success",
    "data": {
        "records": [],
        "total": 100,
        "size": 10,
        "current": 1,
        "pages": 10
    }
}
```

### 1.4 错误码说明

| 错误码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未登录 / Token过期 |
| 403 | 没有权限 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

| icon | String | 否 | 图标名称 |
| sort | Integer | 否 | 排序，默认0 |
| visible | Integer | 否 | 是否显示，默认1 |
| permission | String | 否 | 权限标识（按钮类型时必填） |

**响应示例：**
```json
{
  "code": 200,
  "msg": "新增成功",
  "data": null,
  "timestamp": 1711000000000
}
```

## 三、工单接口

### 3.1 创建工单

**接口地址：** `/office/ticket/create`
**请求方式：** POST
**是否需要认证：** 是
**权限要求：** `ticket:create`

**请求示例：**
```json
{
    "title": "打印机故障",
    "typeId": 1,
    "typeName": "IT支持",
    "content": "三楼会议室的打印机无法使用，显示错误代码E005",
    "priority": 1,
    "attachments": ["https://xxx.com/error.jpg"]
}
```

**参数说明：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| title | String | 是 | 工单标题 |
| typeId | Long | 是 | 工单类型ID |
| typeName | String | 是 | 工单类型名称 |
| content | String | 是 | 工单内容 |
| priority | Integer | 否 | 优先级：1紧急 2普通 3低，默认2 |
| attachments | Array | 否 | 附件URL列表 |

**响应示例：**
```json
{
    "code": 200,
    "msg": "success",
    "data": {
        "id": 1,
        "ticketNo": "TK2024030001",
        "title": "打印机故障",
        "typeName": "IT支持",
        "content": "三楼会议室的打印机无法使用，显示错误代码E005",
        "priority": 1,
        "status": 0,
        "createTime": "2024-03-20 10:30:00"
    },
    "timestamp": 1711000000000
}
```

**status 说明：**

| 值 | 说明 |
|----|------|
| 0 | 待处理 |
| 1 | 处理中 |
| 2 | 已完成 |
| 3 | 已关闭 |
